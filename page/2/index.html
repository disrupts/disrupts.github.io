A Little More About Vim and My Switch to Octopress/Jekyll

May 3rd, 2014

  • More things about vim, update/follow up from last year post on how vim changed my way of understanding computers.
  • my vimrc is up! I finally took the time to choose a method to manage my dotfiles and started using it. Not very much yet up, but used the oportunity to rewrite my messy .vimrc from scratch!
  • I had to start fresh and was a great idea, I only took a few details from the original (such as disabeling arrow keys in insert mode – it forced me to swap from normal mode and insert mode the way it is intended, it was a way of forcing myself to learn!)
  • went through the first chapters of VIMCasts to learn a bit more and do it well!

  • now running vim mainly on a terminal emulator (I don’t see the point anymore of using gvim – aside from preatier fonts and text underlining)

  • so, why moving to octopress? haven’t written yet about that, mainly 2 reasons:

    • wanted to write posts in vim!
    • it alows me to write offline and push changes in a moment!

  • I took me a while to make the switch, I had already tried octopress this summer but never really starting to blog with it and now I see how worthit it is, just great, suits my workflow just so much better than wordPress.

  • Not that WordPress is bad, far from it, the experience was great and would recomended to anyone not wanting to take the time to setup a static blog.

So check out the vimrc, should be pretty strait forward. also took the time to ditch pluggins I was never using (I actually already did that a while back but hadn’t added any). Pathogen looks promising, especially with git submodules! hooray! a simple (though maybe not easy or straitforward at first, eccept you already know a lot about git, which is totally not my case!) way to update plugins! and while keeping them organised!

That should be enough for today, but don’t worry I’ll be gradually (maybe a bit slowly, exams are near) adding more dotfiles to the github repo!

Next will probably be zsh although I should do the same as for vim and take my time to learn a bit more about the tool while rewritting the config file! Should I ditch Oh-my-zsh entirely? or embrace it and set it as a git-submodule too?!
